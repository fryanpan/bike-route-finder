# Project Status — 2026-04-11

## What's Built

### Core Routing
- **Client-side routing engine** using ngraph.path A* on cached Overpass tile data
- Speed-based multimodal costing: Bea bikes preferred paths at 10 km/h, walks unsafe gaps at 1.5-4 km/h
- Per-infrastructure-type speeds for toddler: Fahrradstrasse (12 km/h) > Bike path (10) > Elevated sidewalk (9) > Shared foot path (8) > Residential (4) > Painted bike lane (3)
- Roads without sidewalks excluded entirely in toddler mode
- **Benchmark: 57% avg preferred infrastructure** — beats Valhalla (35%) and BRouter (40%) across 22 Berlin test routes
- Valhalla as fallback when client router can't find a path
- City-agnostic tile prefetch with IndexedDB persistence

### Map & Overlay
- Overpass-based bike infrastructure overlay with per-travel-mode classification
- Text labels on route segments (Radweg, Fahrradstr., etc.) in black with white outline
- Preferred (green) / other (orange) / walking (gray dashed) segment coloring
- Clickable preferred segment suggestions near routes
- Per-region classification rules stored in Cloudflare KV

### Navigation
- Live GPS turn-by-turn with auto-advance (30m proximity trigger)
- Distance-based speech (200m and 50m before turns)
- Current segment indicator (preferred / other / walk)
- Segment feedback (thumbs up/down + report) during navigation

### Admin Tools
- Classification audit: scan cities, group by tag pattern, Mapillary street-level imagery
- Evaluation harness: compare routes across engines
- Route logging to Cloudflare D1
- Per-region classification rules with editable legend items

### Infrastructure
- React + Vite frontend on Cloudflare Pages
- Cloudflare Worker (API proxy, route logging, classification rules)
- Cloudflare KV (classification rules), D1 (route logs, segment feedback)
- Sentry error tracking, Mapillary integration
- 196 tests, CI on every PR

## What's Working Well
- Client router finds Fahrradstrassen and car-free paths that Valhalla misses
- The Overpass overlay and classification system are the single source of truth for routing
- Speed-based costing naturally handles the multimodal problem (walk penalty = slow speed)
- City caching means Berlin tiles load in ~30s first time, ~100ms on return

## Known Issues
1. **Walking segments show 0%** — walk fallback edges may not be connecting properly in the graph
2. **SSE Schwimmhalle route fails** — tile coverage too narrow at northeast Berlin edge
3. **School → Alexanderplatz/Fischerinsel score low** — infrastructure desert near school, not a bug
4. **Travel mode selector still overlaps** on some mobile screen sizes
5. **Valhalla 429s** in benchmark — need longer pauses
